Cheapest Manual Plan (Egypt 2026)
Project target: Nutrition doctor website with appointment scheduling
Model: You build and operate everything manually (custom stack, no WordPress licenses)

====================================================================
1) Assumptions
====================================================================
1. Stack: Django + DRF + React (or Laravel + Blade if you want faster delivery)
2. Hosting: Hetzner VPS (CX22 equivalent)
3. SSL: Let's Encrypt (free)
4. DNS/CDN: optional Cloudflare free plan only (no paid Cloudflare)
5. Email: Brevo free tier (or SMTP from hosting if available)
6. No paid themes, no paid booking plugins, no premium SaaS
7. You do DevOps, development, deployment, and maintenance manually

====================================================================

- VPS: ~200 EGP/month  => ~2,400 EGP/year

- SSL: 0 EGP

Mandatory total per year:
- Minimum: ~2,900 EGP/year

Optional annual add-on range:
- ~600 to 5,400 EGP/year

3) Development Plan (manual execution)
====================================================================
Total timeline: 6 to 8 weeks

Week 1: Infrastructure and foundation
- VPS provisioning
- Nginx + app runtime + database setup
- SSL, firewall, backup cron setup

Week 2: Core backend
- User/doctor/admin roles
- Appointment model and booking rules
- Availability and slot logic

Week 3: Frontend pages
- Home, services, about, contact
- Booking form and schedule calendar
- Responsive mobile UI

Week 4: Integrations
- Email notifications
- WhatsApp click-to-chat
- Maps embed + SEO basics

Week 5: Arabic/English and QA
- i18n for content and forms
- validation, error handling
- browser and mobile testing

Week 6: Launch
- production hardening
- DNS cutover
- monitoring and go-live checklist

Week 7-8 (buffer):
- fixes from real users
- UX polish and performance tuning

====================================================================
4) Monthly manual operations checklist
====================================================================
1. Apply OS and package updates
2. Check SSL renewals and web server logs
3. Verify backups and restore test (at least monthly)
4. Monitor CPU/RAM/disk and DB growth
5. Patch framework dependencies
6. Test booking flow after updates
